Drowsiness Analysis, Prediction and Detection

Drowsiness Dataset

A) For Real Time detection: HaarCascade Frontal Face dataset is used.

HaarCascade is an Object Detection Algorithm used to identify faces in an image or a real time video. The algorithm uses edge or line detection features proposed by Viola and Jones in their research paper “Rapid Object Detection using a Boosted Cascade of Simple Features” published in 2001. The algorithm is given a lot of positive images consisting of faces, and a lot of negative images not consisting of any face to train on them.

Link to the dataset : HaarCascade Frontal Face Dataset

B) For Machine Learning:

We have prepared a dataset using the extracted features from the Real Time Detection of Drowsiness. The extracted features are EAR and MAR. The dataset contains three columns namely Y, EAR and MAR. The size of dataset is 15,952.
We have labeled the data into three categories as 0, 5 and 10. '0' marks as non-drowsy, '5' marks as medium drowsy and '10' marks as severe drowsy. The labelling logic is as follows:
If MAR > 20 then Y = 10
If MAR < 20 and EAR < 0.25 then Y = 10
If MAR < 20 and (EAR >=0.25 and EAR <=0.35) then Y = 5
If MAR < 20 and EAR >0.35 then Y = 0
